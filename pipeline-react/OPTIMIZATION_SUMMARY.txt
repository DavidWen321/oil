## 🎯 性能优化目标

### Core Web Vitals 目标
- FCP (First Contentful Paint) < 1.8s
- LCP (Largest Contentful Paint) < 2.5s
- FID (First Input Delay) < 100ms
- CLS (Cumulative Layout Shift) < 0.1
- TTFB (Time to First Byte) < 800ms

### 其他目标
- Lighthouse 性能评分 > 90
- 首屏加载时间 < 2s（3G 网络）
- 交互响应时间 < 100ms
- 包体积减少 30%+

## 📋 使用指南

### 1. 构建命令

# 开发环境
npm run dev

# 生产构建
npm run build

# 生产构建 + 包体积分析
npm run build:analyze
# 构建完成后打开 dist/stats.html 查看分析报告

# 预览构建结果
npm run preview

### 2. 使用优化组件

# 图片懒加载
import LazyImage from '@/components/common/LazyImage';

<LazyImage
  src="/images/pipeline.jpg"
  alt="管道示意图"
  placeholder="/images/placeholder.svg"
/>

# 虚拟滚动列表
import VirtualList from '@/components/common/VirtualList';

<VirtualList
  data={items}
  itemHeight={50}
  containerHeight={600}
  renderItem={(item, index) => <div>{item.name}</div>}
/>

# 优化的图表组件
import OptimizedChart from '@/components/common/OptimizedChart';

<OptimizedChart
  option={chartOption}
  loading={isLoading}
/>

### 3. 使用防抖/节流

import { useDebounce, useThrottle } from '@/utils/debounce';

// 防抖搜索
const debouncedSearch = useDebounce((query: string) => {
  fetchData(query);
}, 300);

// 节流滚动
const throttledScroll = useThrottle(() => {
  handleScroll();
}, 100);

### 4. 性能监控

性能数据会自动收集并上报到：
- 开发环境：控制台输出
- 生产环境：POST /api/v1/performance

查看当前性能指标：
import { getMetrics } from '@/utils/performance';
console.log(getMetrics());

### 5. 使用 React 性能优化

参考文件: src/examples/performance-optimization.tsx

- 使用 React.memo 避免不必要的重渲染
- 使用 useMemo 缓存计算结果
- 使用 useCallback 缓存函数引用
- 使用唯一 ID 作为列表 key
